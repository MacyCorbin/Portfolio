BookBuster

Teachers select books to assign to students and view each student’s progression through the book. The student view includes the book assignment list and a book view where the student can flip through the pages. Each page is timed for 30 seconds before it is marked as read. The page marking system allows the teacher to view student progress and ensures that the student is not skipping to the end without reading. BookBuster was created by 3 team members: Macy Corbin, Eric Lin, and Peggy Ekwalla.

Programs Used: React, MongoDB, Google Sign In, JavaScript, Node, Illustrator, Photoshop

Macy Corbin: repository organization, register, login, website and logo design

Eric Lin: student website, API, database

Peggy Ekwalla: teacher website

Wizard Duel

User chooses a player and duels their opponents until all are defeated. When the user clicks the cast spell button, the spell is assigned a random value and name. Also, the opponent's counter spell has a random value and name. Each of the spells' values are subtracted from the opposer's health points. The duel continues until every opponent's health reaches 0 or the player's health reaches 0. Wizard Duel was created by Macy Corbin.

Programs Used: JavaScript, JQuery, Bootstrap, HTML, CSS, and Illustrator

Zoom In

Photographers can network and create events through Zoom In. The zip code, photography category, and event description are required to submit an event. An event can only be deleted by the creator. Other photographers can find the events by searching the zip codes close to them and photography category. When a photographer wants to join the event they can add it by clicking the plus symbol. On the events tab, it lists of all the joined events and the minus sign removes the membership. Zoom In was created by 3 team members: Macy Corbin, Eric Lin, and Chris Lane.

Programs Used: Node-JS, Express, Sequelize, bcrypt, Zip Code API, MySQL, JawsDB, and Heroku

Macy Corbin: Front-End

Eric Lin: JavaScript

Chris Lane: Database

Music Match

Music Match searches song titles or artists to find a song and its matching lyrics. Once the criteria is submitted, the top 10 search results from MusiXmatch are retuned. From the list of results, you can click the song you were looking for. The song lyrics are shown and a sample of the song can be played by using the Napster API. The heart icon to the right of selected song saves the song into favorites list. In addition, Music Match will display the current top 5 MusixMatchrated songs. Music Match was created by 4 team members: Macy Corbin, Eric Lin, Chris Lane, and Peggy Ekwalla.

Programs Used: HTML, CSS, Bootstrap, JQuery, Firebase, JQuery-UI, Ajax, MusiXmatch API, and Napster API

Macy Corbin: Design, Layout, HTML, CSS, Bootstrap

Eric Lin: JavaScript, JQuery

Chris Lane: JQuery-UI

Peggy Ekwalla: API’s and Firebase

Process

Ideation to Development

During the ideation process I first look at the problem that needs to be solved and how users can have the best experience when interacting with the final product. I will be using the game I created, Wizard Duel, as an example about how my ideas get to a developed product. The game needed to guide the user through the duel while having a nostalgic theme. I used a white board to visualize the page layout and the effects from interacting with certain elements on the page. While I’m considering these effects I add notes about functions, arrays, variables, and how it would be created in development. My white board is the initial design process that only shows page layouts and basic functionality. If I want a more detailed page design and transition between pages, I use Adobe XD. Then, I add comments, based on my white board notes, in the code to make it structured. Finally, through code iterations a final product is developed.

Theming

With every design I consider the best color and font combination to create a great experience for the user. For the game Wizard Duel, I wanted a nostalgic theme that captures the same tone as the book and movie. I use Adobe Illustrator to explore different color combinations and create graphics. My website and applications share similar colors because I wanted to create my own branding theme.
